프리미엄 디스코드 웹훅 전송기(ui화려하게)

프로그램 제목: HS 웹훅 전송기 v1.1
사용 언어: python

*ui에 대하여
1. 글씨체는 모두 통일, 글씨가 깨지지 않도록.
2. live previews는 실제 디스코드에서 사용하는 이모티콘까지 모두 포함해야함
3. 전체적으로 검은 컨셉을 가져야함.

*보안을 위한 기능
1. 웹훅 링크가 discord로 시작하는지 확인.
2. 라이센스는 첫 로그인 후부터 사용시간을 결정한다.

*유저 페이지
1. 로그인 후 일반 유저가 들어오는 곳 들어오자마자 라이센스에 따른 메시지 개수만큼 사이드바가 생기고, 각 사이드바의 제목은 메시지의 제목으로 한다.(기본값은 메시지 1, 메시지 2와 같은 형식)
2. 메시지
    사이드바를 선택하면 제목과 내용을 입력할 수 있는 공간이 나옴. 제목은 사이드바에 표시될 해당 메시지의 식별 메시지, 실제 전송될 때에는 보여지지 않음.
    내용에서 원하는 내용을 입력하면 실시간으로 우측에 있는 live preview 페이지에 실제 디스코드에서는 어떻게 보내질지 보여줌.(이모티콘 포함)
    내용을 적는 란은 스크롤을 통해 볼 수 있게 하여, live preview란보다는 수직 길이가 짧게 한다. 내용란 아래의 남는 부분에는 주기를 설정할 수 있는 공간과 전송 버튼을 만든다.
    주기는 최소 1일이고, 단위는 일수로만 설정 가능하다. 전송 버튼을 만들어놓는다. 해당 버튼에 마우스를 올리면 (메시지를 전송합니다. 이제부터 ${주기}마다 메시지가 전송됩니다.)라는 글을 보인다.
    메시지 전송 버튼을 통해 메시지를 보내면 마지막 전송시간을 몇월 며칠 몇시 몇분에 마지막으로 보냈는지 표시한다.(연한 글씨로 버튼 아래에) 만약 아직 보낸적이 없다면 메시지 전송 버튼을 통해 메시지 전송을 시작하세요 라는 문구를 적는다.
3. 웹훅 입력란
    메시지와 live preview 아래에 웹훅을 넣을 수 있는 인풋 란을 만든다. 만약 웹훅 전송에 실패했다면 해당 웹훅란을 붉게 바꾸고, 웹훅 실행 실패라 뜨게 한다. 웹훅 입력 시에 디스코드 웹훅이 맞는지 링크 앞부분을 검증한다.(아니라면 입력란 전체를 연하게 바꾸고 입력란 아래에 작은 글씨로 디스코드 웹훅이 아니다 라는 경고문 표시)
4. 자동저장
    메시지와 해당 메시지에 대한 웹훅은 자동저장하되, 만약 라이센스가 끊겼다가 재등록했을 때 기존 라이센스보다 메시지 개수나 웹훅 개수가 적다면, 해당 란은 수정할 수 없도록 잠그되, 칸에 저장된 데이터는 버리지 않는다.
5. 로그아웃 버튼
    사이드바 가장 아래에 라이센스 정보(보낼 수 있는 메시지 종류 수, 메시지당 사용할 수 있는 웹훅 수, 남은 일수(만약 하루 이하로 남았다면 몇시간 몇분 남았는지 표시))

*관리자 페이지
1. 라이센스 생성
    관리자로 로그인하면 며칠동안 사용할지, 메시지당 지정할 수 있는 웹훅 개수는 몇개인지, 메시지 종류는 몇개 지정할지 선택할 수 있는 창을 만들고 라이센스 생성 버튼을 눌러 HS로 시작하는 임의의 8자리 문자열을 만들고, 버튼을 통해 클립보드에 복사할 수 있게 한다.
2. 상태
    라이센스를 관리할 수 있는 창이다. 현재 사용중인 모든 라이센스를 보여주고, 만료된 내역, 사용중인 내역, 아직 미사용중인 발급된 내역을 필터링하여 볼 수 잇게 한다. 
    만료된 내역: 일괄 삭제 버튼을 만든다.
    사용중인 내역: 목록 형태로 표시하고, 라이센스명, 남은 기간을 표시한다.
    미사용중인 내역: 목록 형태로 표시하되, 각 라이센스마다 개별 삭제 버튼을 만들어 삭제할 수 있게 하고, 전체 일괄 삭제 버튼도 만든다.
3. 로그아웃 버튼
    사이드바(라이센스 생성, 상태) 아래에 로그아웃 버튼을 만든다.

*로그인 페이지
1. 자동 로그인
    만약 라이센스가 만료되지 않은 상태에서 프로그램을 껏다가 켜면 자동으로 로그인한다.(로그인 화면에서 자동 로그인 체크박스에 체크를 해놓은 경우)
